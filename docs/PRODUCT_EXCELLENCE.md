# Product Excellence

> Padrao minimo esperado para produtos criados pelo framework.

## Dimensoes

Todo produto deve buscar excelencia em:

- UX
- UI
- Mobile
- Desktop
- Performance
- Conversao
- Clareza
- Produtividade
- Escalabilidade
- Consistencia

## Product Audit

Categorias com nota de 0 a 10:

- Copy
- Marketing
- Brand
- Comunicacao
- Conversao
- SEO
- Assets
- Landing Page
- Design System
- Product Market Fit
- Escalabilidade

Se qualquer categoria ficar abaixo de 8, o framework deve sugerir melhorias automaticamente e acionar `product-evolution-planner` quando houver auditoria.

## Modos visuais

| Modo | Uso |
|------|-----|
| Legacy Mode | Projeto existente com padroes fortes que devem ser preservados |
| Hybrid Mode | Projeto existente com espaco para modernizacao incremental |
| Greenfield Mode | Projeto novo ou sem identidade definida |
| Automatic Mode | Orchestrator decide por escopo, risco e contexto |
