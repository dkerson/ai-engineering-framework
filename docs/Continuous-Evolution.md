# Continuous Evolution

> Capacidade de gerar oportunidades futuras ao fim de cada mission.

## Objetivo

Transformar entregas em aprendizado e melhoria continua.

## Ao finalizar uma mission

Identificar automaticamente:

- Oportunidades futuras
- Melhorias
- Refatoracoes
- Features
- Dividas tecnicas
- Melhorias de UX
- Melhorias de produto
- Melhorias de growth/brand

## Backlog sugerido

Cada item deve conter:

- Descricao
- Tipo
- Impacto
- Esforco
- Risco
- Prioridade
- Dependencias

## Regra

Continuous Evolution e sugestivo. O Orchestrator nao deve executar novos itens sem uma nova mission ou autorizacao clara do usuario.
