# Checklist — Qualidade SQL

- [ ] SELECT explícito (evitar SELECT *)
- [ ] Aliases legíveis
- [ ] JOINs com condição completa
- [ ] Filtros no lugar certo (WHERE vs HAVING)
- [ ] Sem duplicidade por join incorreto
- [ ] NULLs tratados explicitamente
- [ ] Comentário em lógica não óbvia
- [ ] Compatível com `rules/data/sql-standards.md`
