# Knowledge Quality Checklist

- [ ] Metadata is complete.
- [ ] Canonical identity is defined.
- [ ] Content hash is recorded.
- [ ] Source origin is recorded.
- [ ] No unauthorized secrets or PII are indexed.
- [ ] Version is documented.
- [ ] Permissions are defined.
- [ ] Content was reviewed.
- [ ] Duplicate title/hash clusters were evaluated.
- [ ] Lexical and vector index status is known.
- [ ] Curation flow exists for unanswered questions.
